# 수동 테스트 가이드

자동화로 커버하기 어려운 영역에 대한 수동 테스트 가이드입니다.

## 자동화 vs 수동 테스트

| 테스트 유형 | 자동화 권장 | 수동 권장 |
|-----------|------------|----------|
| 기능 회귀 테스트 | O | |
| 크로스 브라우저 기본 | O | |
| 모바일 웹 에뮬레이션 | O | |
| **실제 모바일 기기** | | O |
| **네이티브 앱** | | O |
| **UX/접근성 검증** | | O |
| **탐색적 테스트** | | O |

---

## 모바일 실기기 테스트

### 왜 필요한가?

Playwright 에뮬레이션은 viewport, user-agent 등을 시뮬레이션하지만 다음은 재현이 어렵습니다:

- iOS Safari 특유의 버그
- 터치 제스처 (스와이프, 핀치줌 등)
- 기기별 성능 차이
- 네트워크 상태 변화 (3G/LTE/WiFi 전환)

### 테스트 체크리스트

#### iOS Safari

- [ ] 상단 주소창/하단 툴바 동작
- [ ] 키보드 올라올 때 레이아웃
- [ ] 스크롤 바운스 효과
- [ ] 줌 동작
- [ ] 100vh 문제 (safe-area)

#### Android Chrome

- [ ] 뒤로가기 버튼 동작
- [ ] 키보드 올라올 때 입력창 가림
- [ ] 가로/세로 전환
- [ ] 저사양 기기 성능

### 권장 테스트 기기

| 플랫폼 | 기기 | 비고 |
|-------|------|------|
| iOS | iPhone 14/15 | 최신 iOS |
| iOS | iPhone SE | 작은 화면 |
| Android | Pixel | 표준 Android |
| Android | Samsung Galaxy | 국내 점유율 높음 |

## 네이티브 앱 테스트

Playwright는 웹 브라우저 전용입니다. 네이티브 앱 테스트는 다른 도구가 필요합니다.

### 대안 도구

| 도구 | 용도 |
|------|------|
| **Appium** | iOS/Android 앱 자동화 |
| **XCUITest** | iOS 전용 |
| **Espresso** | Android 전용 |
| **Maestro** | 간단한 모바일 앱 테스트 |

### 수동 테스트 포인트

- [ ] 앱 설치/업데이트/삭제
- [ ] 푸시 알림
- [ ] 백그라운드/포그라운드 전환
- [ ] 메모리 부족 상황
- [ ] 네트워크 끊김/복구

---

## UX/접근성 테스트

### 수동 확인 항목

- [ ] 탭 순서가 논리적인가?
- [ ] 색맹 사용자도 구분 가능한가?
- [ ] 화면 리더기로 읽히는가?
- [ ] 키보드만으로 조작 가능한가?
- [ ] 터치 타겟 크기가 충분한가? (44x44px 이상)

### 도구

| 도구 | 용도 |
|------|------|
| axe DevTools | 접근성 자동 검사 |
| VoiceOver (Mac) | 화면 리더기 |
| TalkBack (Android) | 화면 리더기 |
| Contrast Checker | 색상 대비 검사 |

## 탐색적 테스트

정해진 시나리오 없이 자유롭게 테스트하는 방법입니다.

### 언제 하는가?

- 새 기능 출시 전
- 버그 재현이 어려울 때
- 자동화 시나리오 발굴 시

### 방법

1. **타이머 설정**: 30분~1시간
2. **목표 설정**: "결제 플로우 깨뜨리기"
3. **기록**: 발견한 이슈 즉시 기록
4. **결과 공유**: 팀과 발견 사항 공유

### 탐색 아이디어

- 극단적인 입력값 (빈값, 매우 긴 텍스트, 특수문자)
- 빠른 연타 클릭
- 브라우저 뒤로가기/새로고침
- 네트워크 끊기/복구
- 다른 탭에서 로그아웃 후 동작

## 수동 테스트 기록 템플릿

```markdown
## 테스트 세션

- 날짜: 2025-01-02
- 테스터: 홍길동
- 환경: iPhone 14, iOS 17.2, Safari
- 목표: 결제 플로우 검증

## 발견 사항

### 이슈 1
- 위치: 결제 확인 페이지
- 현상: 카드 번호 입력 시 키패드가 화면 가림
- 재현: 항상
- 심각도: 중

### 이슈 2
- 위치: 주문 완료 페이지
- 현상: 뒤로가기 시 결제가 중복됨
- 재현: 가끔
- 심각도: 상
```

## 자동화와 수동의 조합

이상적인 QA 프로세스:

1. **자동화**: 회귀 테스트, 크로스 브라우저 기본
2. **수동**: 새 기능, 실기기, UX
3. **탐색적**: 정기적으로 (스프린트당 1회)

```
┌─────────────────────────────────────┐
│ 배포 전 QA 체크리스트                  │
├─────────────────────────────────────┤
│ [x] 자동화 테스트 통과                  │
│ [ ] 실기기 테스트 (iOS Safari)         │
│ [ ] 실기기 테스트 (Android Chrome)     │
│ [ ] 탐색적 테스트 30분                 │
└─────────────────────────────────────┘
```
