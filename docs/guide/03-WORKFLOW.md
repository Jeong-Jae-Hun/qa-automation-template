# 일일 워크플로우

QA 테스트 작업의 일반적인 흐름입니다.

## 워크플로우 개요

```
1. 테스트 케이스 정의
   └─ 자연어로 테스트 설명

2. Claude가 스크립트 생성
   └─ 자동으로 파일 생성

3. 테스트 실행
   └─ /qa run-test

4. 결과 확인 및 수정
   └─ 실패 시 수정 요청

5. 커밋
   └─ git commit
```

## 상세 단계

### 1. 테스트 케이스 정의

테스트하고 싶은 내용을 자연어로 작성합니다:

```
결제 페이지에서:
1. 신용카드 정보 입력
2. 결제 버튼 클릭
3. 결제 완료 메시지 확인
```

### 2. Claude에게 요청

```bash
claude
```

```
결제 페이지에서 신용카드로 결제하면
결제 완료 메시지가 표시되는지 테스트해줘
```

### 3. 테스트 실행

```bash
# 특정 그룹 실행
/qa run-test checkout

# 전체 실행
/qa run-test
```

### 4. 결과 분석

**성공 시:**
```
 PASS  checkout/payment.spec.ts
   ✓ 신용카드 결제 완료 (2.3s)
```

**실패 시:**
```
 FAIL  checkout/payment.spec.ts
   ✗ 신용카드 결제 완료
     Error: 요소를 찾을 수 없음: .payment-success
```

### 5. 실패 시 수정

Claude에게 수정 요청:

```
결제 완료 메시지가 .success-message 클래스에 있어. 수정해줘
```

### 6. 커밋

테스트가 완료되면 저장:

```bash
git add .
git commit -m "결제 테스트 추가"
git push
```

## 자주 사용하는 패턴

### 새 기능 테스트 추가

```
/qa new-test 상품 검색하면 결과 목록이 표시되는지 확인
```

### 기존 테스트 수정

```
login.spec.ts에서 에러 메시지 확인하는 테스트 추가해줘
```

### 브라우저로 디버깅

```
결제 테스트를 브라우저 보면서 실행해줘
```

Claude가 실행:
```bash
npm run test:headed -- tests/checkout/
```

### 스크린샷 확인

실패한 테스트의 스크린샷 확인:

```
reports/screenshots/ 폴더에 스크린샷 있어?
```

## 팁

1. **작은 단위로 테스트**: 한 테스트에 하나의 기능만
2. **실패 시 바로 수정**: 실패 원인을 바로 파악하고 수정
3. **자주 커밋**: 테스트가 통과할 때마다 커밋
