# VS Code + Claude 설정 가이드

> 가장 접근하기 쉬운 QA 자동화 환경

---

## 왜 VS Code인가?

| 도구 | 장점 | 단점 |
|------|------|------|
| **VS Code + Claude** | GUI 친화적, 시각적 피드백 | 설정 약간 필요 |
| Claude Code CLI | 빠름, 파워유저용 | 터미널 익숙해야 함 |
| Cursor | AI 내장 | 유료, 학습 필요 |

**권장**: VS Code + Claude 확장 (무료, 쉬움)

---

## 1. VS Code 설치

### 다운로드

https://code.visualstudio.com 에서 다운로드 후 설치

### 한국어 설정

1. `Cmd + Shift + P` (맥) 또는 `Ctrl + Shift + P` (윈도우)
2. "Configure Display Language" 검색
3. "한국어" 선택
4. VS Code 재시작

## 2. 필수 확장 프로그램

### 설치 방법

1. 왼쪽 사이드바에서 **확장** 아이콘 클릭 (또는 `Cmd+Shift+X`)
2. 검색창에 이름 입력
3. **Install** 클릭

### 필수 확장

| 확장 | 설명 |
|------|------|
| **Playwright Test for VSCode** | 테스트 실행/디버깅 |
| **Claude** | Claude AI 연동 |
| **Korean Language Pack** | 한국어 인터페이스 |

### 권장 확장

| 확장 | 설명 |
|------|------|
| **GitLens** | Git 기록 시각화 |
| **Error Lens** | 에러 인라인 표시 |
| **Prettier** | 코드 자동 포맷 |

---

## 3. Claude 확장 설정

### 설치

1. 확장에서 "Claude" 검색
2. Anthropic 공식 확장 설치
3. VS Code 재시작

### 연동

1. 왼쪽 사이드바에서 Claude 아이콘 클릭
2. "Sign in" 클릭
3. Anthropic 계정으로 로그인

### 사용법

1. **채팅 패널** 열기: Claude 아이콘 클릭
2. **파일 컨텍스트**: 파일 선택 후 Claude에게 질문
3. **코드 생성**: 자연어로 요청

---

## 4. Playwright 확장 설정

### 설치 후 설정

1. 확장에서 "Playwright Test for VSCode" 설치
2. VS Code 재시작
3. 왼쪽에 **테스트 아이콘** (플라스크 모양) 생김

### 테스트 탐색기 사용

1. **테스트 아이콘** 클릭
2. 테스트 목록 확인
3. **▶** 버튼으로 실행

### 디버깅

1. 테스트 옆 **▶** 오른쪽 클릭
2. "Debug Test" 선택
3. 브레이크포인트 설정하여 단계별 실행

---

## 5. 프로젝트 열기

### 폴더 열기

1. `파일 > 폴더 열기` (또는 `Cmd+O`)
2. `qa-automation-template` 폴더 선택
3. "이 폴더의 작성자를 신뢰하시겠습니까?" → **예**

### 작업 영역 저장 (선택)

1. `파일 > 작업 영역을 다른 이름으로 저장`
2. 다음에 바로 열 수 있음

---

## 6. 테스트 실행하기

### 방법 1: 테스트 탐색기

1. 왼쪽 **테스트 아이콘** 클릭
2. 실행할 테스트 찾기
3. **▶** 클릭

### 방법 2: 파일에서 직접 실행

1. 테스트 파일 열기 (`.spec.ts`)
2. `test` 위에 표시되는 **▶ Run** 클릭

### 방법 3: 터미널

1. `터미널 > 새 터미널` (또는 `` Ctrl+` ``)
2. 명령어 입력:

```bash
npm test
```

---

## 7. Claude로 테스트 작성하기

### 새 테스트 요청

1. Claude 채팅 패널 열기
2. 자연어로 요청:

```
로그인 페이지에서 잘못된 비밀번호 입력하면
에러 메시지가 표시되는지 테스트해줘
```

3. Claude가 코드 생성
4. "파일에 적용" 클릭

### 기존 테스트 수정

1. 테스트 파일 선택
2. Claude에게 요청:

```
이 테스트에서 비밀번호가 빈 값일 때 케이스도 추가해줘
```

### 에러 수정

1. 실패한 테스트의 에러 메시지 복사
2. Claude에게:

```
이 에러가 발생했어:
[에러 메시지 붙여넣기]
어떻게 수정해야 해?
```

## 8. Git 사용하기 (GUI)

### Source Control 패널

1. 왼쪽 **소스 제어** 아이콘 클릭 (가지 모양)
2. 변경된 파일 목록 확인

### 커밋하기

1. 변경 파일 옆 **+** 클릭 (Stage)
2. 메시지 입력란에 커밋 메시지 작성
3. **✓** 버튼 클릭 (Commit)

### Push하기

1. 하단 상태바에서 **↑** 아이콘 클릭
2. 또는 Source Control의 **...** → **Push**

### 브랜치 변경

1. 하단 상태바에서 현재 브랜치 이름 클릭
2. 목록에서 브랜치 선택 또는 새로 생성

## 10. 추천 설정

### settings.json

`Cmd+,` → 오른쪽 상단 **{}** 아이콘 클릭:

```json
{
  "editor.formatOnSave": true,
  "editor.tabSize": 2,
  "files.autoSave": "afterDelay",
  "playwright.reuseBrowser": true,
  "playwright.showTrace": true
}
```

### 자동 저장

- `파일 > 자동 저장` 체크

### 폰트 크기

- `Cmd+` / `Cmd-` 로 조절

## 문제 해결

### Claude 확장이 안 보임

1. 확장이 설치되었는지 확인
2. VS Code 재시작
3. 확장이 활성화되었는지 확인

### Playwright 테스트가 안 보임

1. 프로젝트 폴더가 열려있는지 확인
2. `node_modules` 폴더가 있는지 확인
3. 없으면 터미널에서: `npm install`

### 터미널에서 명령어가 안 됨

1. `터미널 > 새 터미널`
2. 올바른 폴더인지 확인
3. `pwd` 입력하여 현재 경로 확인
